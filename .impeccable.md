# Impeccable — Air Quality Monitor (Frontend)

Project-specific design guidance for AI-assisted UI work in `air-qua-monitor-front`.

## Design Context

### Users

- **Public visitors** checking live air quality in Addis Ababa on the landing page and map — often on mobile, wanting a quick read of AQI and PM levels near them.
- **Registered users** (citizens, researchers) who sign in to explore station history, compare sensors, and export data.
- **Administrators** managing users, locations, and devices.

**Job to be done:** Understand current air quality at a glance, drill into a specific station, and (for power users) analyze trends over time.

**Context:** Map-first for discovery; station cards and popups for quick readings; `/sensors` for deep analytics.

### Brand Personality

**Clear · Grounded · Trustworthy**

The product should feel like a credible environmental health tool — not a generic SaaS dashboard. Data (AQI, PM2.5, PM10) is the hero. Motion and color reinforce meaning (green = good air), not decoration.

**Emotional goals:** Confidence in the data, calm when air is good, appropriate urgency when unhealthy — without alarmism.

### Aesthetic Direction

- **Visual tone:** Clean, map-centric, lightly technical. Frosted glass chrome on the dashboard map; structured AppShell (sidebar + header) for authenticated app pages.
- **Color system:** Green primary (`#16a34a`), brand blue for CTAs (`#016FC4`), EPA-style AQI colors on markers and cards (green / yellow / orange / red / purple). Muted surfaces use pale green tints, not flat gray.
- **Typography:** Geist sans; bold station names; tabular nums for metrics; uppercase micro-labels for pollutant names.
- **References:** WAQI/AQICN-style AQI hero blocks; IQAir-like compact station cards; Microsoft Weather–style color-coded severity.
- **Anti-references:** Purple-blue AI gradients, bounce animations, tall narrow mobile-card popups with empty whitespace, rainbow charts without semantic meaning.
- **Theme:** Light mode primary; dark tokens exist but light is the default experience.

### Design Principles

1. **Data first** — AQI and PM values are the largest, most colorful elements; chrome stays quiet.
2. **One visual language for air quality** — Map dots, popups, station cards, and badges share the same EPA color scale and level names.
3. **Progressive depth** — Map popup → station browser (`/stations`) → full analytics (`/sensors`); each step adds detail without repeating the same layout.
4. **Purposeful motion** — Entrance fades, sensor-pulse loading, hover lift on cards; always respect `prefers-reduced-motion`.
5. **Accessible contrast** — Status must not rely on color alone (include labels); WCAG AA for text; yellow AQI blocks use dark text.

### Key Routes & Patterns

| Route | Purpose |
|-------|---------|
| `/` | Public landing + embedded map + station carousel |
| `/dashboard` | Fullscreen map + floating MapPageChrome |
| `/stations` | Browse all sensors, latest readings, pick a station |
| `/sensors` | Authenticated charts, filters, CSV export |
| `/settings`, `/admin` | AppShell layout |

### Components to Reuse

- `LoadingState` — branded loading (sensor pulse rings)
- `AppShell` + `AppHeader` + `Sidebar` — authenticated pages
- `StationExplorerCard` / map popup — shared AQI hero + metrics strip
- `fetchPublicDashboardData()` — single source for public station list

### Accessibility

- Keyboard-focusable station cards and popup actions
- `aria-live` on loading states
- Reduced motion: disable orbit/pulse animations
- Minimum touch targets on map chrome and CTAs
